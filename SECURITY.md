# Security Policy

## Supported Versions

Security fixes are applied to the latest version of each project only.

| Status    | Supported |
| --------- | --------- |
| Latest    | ✅        |
| Older     | ❌        |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Use **GitHub Private Vulnerability Reporting** on the affected repository:

1. Go to the repository's **Security** tab
2. Click **"Report a vulnerability"**
3. Fill in the details and submit

GitHub will keep the report confidential until a fix is released.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact and severity
- Any suggested mitigations (optional)

### Response timeline

| Stage               | Target      |
| ------------------- | ----------- |
| Acknowledgement     | 3 days      |
| Status update       | 7 days      |
| Fix or disclosure   | 90 days     |

Reports are handled on a best-effort basis. StangaNet is a personal organization with no dedicated security team.
