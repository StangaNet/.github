# Support

## How to Get Help

StangaNet projects are personal works maintained by a single developer.
Support is offered on a best-effort basis.

### Before opening an issue

- Read the project's `README.md` thoroughly
- Check existing [Issues](../../issues) to see if your question was already answered
- Search [Discussions](../../discussions) if enabled on the repository

### Where to ask

| Channel              | Use for                                        |
| -------------------- | ---------------------------------------------- |
| **GitHub Issues**    | Bug reports (use the template)                 |
| **GitHub Discussions** | Questions, ideas, general conversation       |

### What not to expect

- Real-time support or SLA guarantees
- Feature development on demand
- Support for unsupported or outdated versions

### Security issues

Do **not** post security vulnerabilities publicly.
See [SECURITY.md](./SECURITY.md) for the responsible disclosure process.
